API

All endpoints below are served by this process. Paths are relative to this host. Readings are self-describing: each carries the meter's function (device string, e.g. VOLT) and unit, correct even if the front-panel mode changed since the previous call.

Machine-readable spec

This service is FastAPI, so a full OpenAPI schema and interactive docs are always available — point AI agents or codegen at these:

GET/openapi.jsonOpenAPI 3 schema (JSON)
GET/docsSwagger UI (interactive)
GET/redocReDoc (reference)

Reading the meter

GET /api/status

One-call snapshot for HTTP pollers: current value, mode, full config, and a time-smoothed value. Recommended for integrations (e.g. Home Assistant).

Query: seconds (float, default 60) — smoothing window. Must be > 0.

Taking the reading also records a sample, so a client that polls /api/status keeps its own smoothing window fresh with no live viewer needed. When persistence is disabled, smoothed.samples is 0 and smoothed.value is null.

{% raw %}
$ curl 'http://HOST/api/status?seconds=60'
{
  "timestamp": 1719000000.12, "function": "VOLT", "value": 4.567, "unit": "V",
  "state": {"function": "VOLT_DC", "rate": "MEDIUM", "auto_range": true, "range": "5V"},
  "smoothed": {"function": "VOLT", "unit": "V", "value": 4.55, "samples": 12,
               "window_seconds": 60, "min": 4.51, "max": 4.59}
}
{% endraw %}

GET /api/measurement

A single fresh reading taken on demand. Also records the sample when persistence is enabled.

{% raw %}
{"timestamp": 1719000000.12, "function": "VOLT", "value": 4.567, "unit": "V"}
{% endraw %}

GET /api/measurement/smoothed

Just the time-smoothed value: the mean of recorded readings for a function over the window, with sample count and min/max. Requires persistence (503 otherwise).

Query: seconds (float, default 60, must be > 0); function (optional enum name, e.g. RESISTANCE; defaults to the meter's current function).

{% raw %}
$ curl 'http://HOST/api/measurement/smoothed?seconds=30'
{"function": "VOLT", "unit": "V", "value": 4.55, "samples": 6,
 "window_seconds": 30, "min": 4.51, "max": 4.59}
{% endraw %}

GET /api/state

The meter's current configuration (read through to the device, never cached).

{% raw %}
{"function": "VOLT_DC", "rate": "MEDIUM", "auto_range": true, "range": "5V"}
{% endraw %}

GET /api/identify

The meter's *IDN? identity string.

{% raw %}
{"identity": "OWON,XDM1041,..."}
{% endraw %}

GET /api/functions

All selectable measurement functions and their human labels.

{% raw %}
[{"name": "VOLT_DC", "label": "DC Voltage"}, {"name": "RESISTANCE", "label": "Resistance"}, ...]
{% endraw %}

GET /api/history

Persisted readings, oldest first. History is recorded while a dashboard/WebSocket client is watching, and whenever /api/status or /api/measurement is called. Requires persistence (503 otherwise).

Query: since / until (Unix seconds); function (enum name, e.g. RESISTANCE); limit (int, default 1000 — most recent N).

{% raw %}
[{"timestamp": 1719000000.12, "function": "VOLT", "value": 4.567, "unit": "V"}, ...]
{% endraw %}

Controlling the meter

All control endpoints return the updated state object (same shape as /api/state).

POST /api/function

Switch the active measurement function.

Body (JSON): function (enum name, required); rtd_type (string, default "KITS90", for temperature).

{% raw %}
$ curl -X POST http://HOST/api/function -H 'Content-Type: application/json' \
       -d '{"function": "RESISTANCE"}'
{% endraw %}

POST /api/rate

Set the measurement rate.

Body (JSON): rate — one of SLOW, MEDIUM, FAST.

POST /api/auto-range

Enable auto-ranging. No body.

Live stream

WS /ws/live

WebSocket of live readings, one JSON message per poll interval. Connecting starts the on-demand poller; it stops when the last client disconnects.

{% raw %}
{"timestamp": 1719000000.12, "function": "VOLT", "value": 4.567, "unit": "V"}
{% endraw %}

Errors

Unknown function/rate names return 422. Endpoints that need stored history (/api/history, /api/measurement/smoothed) return 503 when persistence is disabled.
