SCPI

This server exposes the meter on the network as a raw-socket SCPI instrument, so tools like pyvisa, NI-VISA, and sigrok can drive the XDM1041 over TCP. The SCPI socket, the web UI, and the live poller all share one arbitrated connection to the meter, so multiple clients can connect at once — commands are serialised, never interleaved mid-transaction.

Query vs. write: a line containing ? is treated as a query and its reply is written back (newline-terminated); any other line is a write with no reply. This matches how pyvisa distinguishes query() from write(). The front panel stays usable while remote commands work.

Connect with pyvisa

{% raw %}
import pyvisa
rm = pyvisa.ResourceManager()
{% endraw %}dmm = rm.open_resource("{{ resource }}", read_termination="\n", write_termination="\n"){% raw %}
print(dmm.query("*IDN?"))      # identify
print(dmm.query("MEAS1?"))     # current reading (unitless float)
dmm.write("CONF:VOLT:DC")      # switch to DC volts
{% endraw %}

XDM1041 commands

The command set below is what this meter understands (reverse-engineered from TheHWcave/OWON-XDM1041). Send any of these verbatim over the socket.

Queries (return a value)

*IDN?Identification string
MEAS1?Current reading as a bare float — no unit; derive it from the function
FUNC1?Active function, wrapped in literal double-quotes (e.g. "VOLT AC")
RATE?Measurement rate: S, M, or F
AUTO?Auto-range enabled (1/0)
RANGE?Current range string. Not answered in DIOD/CONT/TEMP modes.

Function select (writes)

CONF:VOLT:DCDC voltage (V)
CONF:VOLT:ACAC voltage (V)
CONF:CURR:DCDC current (A)
CONF:CURR:ACAC current (A)
CONF:RESResistance (Ω)
CONF:CAPCapacitance (F)
CONF:FREQFrequency (Hz)
CONF:PERPeriod (s)
CONF:DIODDiode (V)
CONF:CONTContinuity (Ω)
CONF:TEMP:RTD TYPETemperature (°C); TYPE is KITS90 (K-type) or PT100

Other writes

RATE RSet rate; R is S (slow), M (medium), or F (fast)
AUTOEnable auto-ranging

Note: MEAS1? returns an unformatted float with no unit (the unit follows from the active function — see the API page's unit field), and FUNC1? wraps its answer in double-quotes. Prefer the HTTP API if you want unit-tagged, self-describing readings.
